MatrixDice is an experimental mathematical puzzle framework that combines linear algebra, matrix theory, and cube-based puzzle mechanics.

Instead of traditional numbered faces, each face of the cube contains a 3×3 integer matrix. Every matrix satisfies a collection of mathematical constraints, with the determinant of each face corresponding to its face number (1–6). The project explores how matrix invariants and cube transformations can form the basis of a new class of mathematical puzzles.

Current features include:

1)Determinant-constrained matrix generation

2)Multiple matrix generation strategies

3)Matrix validation and integrity checks

4)Cube rotation mechanics

5)Matrix invariant computation (determinant, rank, trace, eigenvalues, condition number)

6)JSON serialization and deserialization

7)Comprehensive self-testing framework

8)Extensible object-oriented architecture

Roadmap

Future versions aim to introduce:

1)Tower of Hanoi-inspired puzzle mechanics

2)Matrix transformation moves

3)AI solvers (BFS, DFS, A*, IDA*, MCTS, Reinforcement Learning)

4)State-space exploration

5)3D visualization

6)Interactive GUI

7)Research tools for computational and recreational mathematics

Motivation The idea for MatrixDice originated from a curiosity sparked by the Tower of Hanoi puzzle while studying recursion in high school. The project explores whether concepts from linear algebra, group theory, and algorithm design can be combined into a novel mathematical puzzle.
